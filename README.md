# Защищенное хранилище файлов

Версия приложения: `0.12.8`.

SecureVault - кроссплатформенный защищенный проводник для Windows, Android и Linux. Проект ведется как защищенный файловый менеджер с просмотром обычных и зашифрованных файлов, медиа-плеером, редакторами, сетевыми/облачными расположениями, плагинами и переносимыми настройками.

## Полный целевой функционал

- Проводник с расположениями слева, файлами/папками по центру и скрываемой областью просмотра справа; ширина областей меняется вручную.
- Расположения: диски Windows, корень Linux/Android, стандартная память телефона Android, скрытое хранилище программы, ZIP/RAR-архивы как папки, torrent-файлы как папки, SMB, FTP, SFTP/SSH, WebDAV/облачные плагины, составные RAID0/RAID1-расположения.
- Загрузка/выгрузка файлов с выбором режима plain/encrypted и удалением источника после операции.
- Шифрование/расшифрование файлов общим или уникальным ключом, выбор алгоритма, ключ строкой или файлом, автоматическое расшифрование имен файлов и папок при доступном ключе.
- Настройки пароля входа, отдельного пароля зашифрованных файлов, grace-периода, сохраненных паролей, очистки после ошибок входа и защиты экрана.
- Зашифрованный конфигурационный файл, экспорт конфигурации, языков, плагинов и служебных данных.
- Недавние и избранные файлы/папки с отдельными лимитами, проверкой доступности и контекстным меню.
- Контекстные меню для файлов, папок, пустой области и выделения: открыть, создать, создать папку, копировать, вырезать, вставить, удалить, переименовать, свойства, отправить, ZIP, шифрование, расшифрование, использование папки как медиа-источника.
- Создание обычных и зашифрованных файлов, CSV, изображений и папок.
- Поиск по имени и содержимому, регулярные выражения, рекурсивный режим, фильтры размера и дат.
- Просмотр текста, HTML, документов, CSV, изображений, аудио, видео и электронных книг; неизвестные расширения предлагаются к открытию во внешней программе.
- Текстовый редактор с режимами текст/HEX/binary, поиском и конвертацией кодировок.
- Табличный просмотр CSV/TSV и обычный просмотр таблиц как файла.
- Чтение электронных книг с TTS устройства.
- Просмотр изображений стрелками/свайпами, полноэкранный просмотр, корректное масштабирование.
- Галерея как на телефоне: фото и видео по годам/месяцам, миниатюры, масштабирование до календаря.
- Музыка и Видео как медиатеки: вкладки Текущие, Все, Плейлисты, Альбомы, Исполнители, Жанры, Папки, Последние предыдущие.
- Медиа-плеер: плейлист снизу, следующий/предыдущий трек, shuffle, repeat-one, продолжение с позиции остановки, скорость, субтитры, выбор аудиодорожки, lyrics, обложки аудио и кадры видео.
- Мини-плеер для фонового аудио/видео внутри окна приложения, управление аппаратными media-клавишами/гарнитурой, настройки фонового поведения.
- Редакторы изображений, аудио и видео отдельными окнами: слои, crop/rotate/draw, waveform/timeline trim, склейка/микс/качество через ffmpeg.
- Плагины: ZIP-установка, удаление, экспорт, информация, общие настройки, отдельные профили подключений к расположениям, версии, repository/update URL, proxy, OS/arch components, capabilities, cloud/network/media executors.
- Предустановленные реальные плагины: Яндекс.Диск WebDAV, Nextcloud WebDAV, FTP, embedded SFTP/SSH, embedded SMB2/3, RAR Archive Support, RAID0/RAID1 virtual locations.
- Torrent-раздел: добавление torrent, хранение в папке программы, просмотр как папки, выбор файлов, потоковое воспроизведение аудио/видео через локальный HTTP Range-stream и продолжение с сохраненной позиции.
- Фоновые операции шифрования/расшифрования с нижней панелью прогресса, сворачиванием и отменой.
- Локализация: русский и английский по умолчанию, установка языковых файлов, экспорт английского образца.
- Android: запрос прав на файлы/медиа с запоминанием отказа, повторный запрос из настроек, компактная верхняя панель без дублирования названия программы.

## Реализовано в 0.12.8

- Версия повышена до `0.12.8` в `pubspec.yaml`, коде приложения и README.
- В разделах `Музыка` и `Видео` плеер больше не встраивается в список медиатеки: первый выбранный файл открывается во все окно, а если пользователь уже свернул сессию в мини-плеер, следующий выбранный плейлист/папка переключается в мини-плеере без принудительного разворачивания.
- Вкладка `Текущие` для музыки и видео теперь показывает весь текущий плейлист/папку/список из интернет-плагина, а не только активный файл.
- Менеджер плагинов получил кнопку включения/отключения плагина; отключенные плагины исключаются из runtime, расположений, media-разделов и fileHandler-обработки.
- Для онлайн-музыки действие `Копировать` теперь подготавливает загрузку трека: при `Вставить` в выбранную папку файл скачивается туда, включая виртуальные remote-расположения через их executor.
- Поиск в разделе `Hitmoz` и других plugin media sections применяет результат сразу после кнопки `Применить`; фильтры в таких онлайн-разделах скрыты, потому что локальные фильтры размера/дат неприменимы к удаленному каталогу.
- Мини-плеер показывает название текущего трека бегущей строкой вместо `SecureVault media`; бесполезная кнопка дополнительного сворачивания удалена.
- Текстовый редактор исправлен: поиск прокручивает к найденной строке, HEX/binary режимы показывают ASCII-колонку справа с учетом выбранной кодировки, область текста растягивается по ширине окна.
- В контекстное меню файлов добавлена команда `Редактировать`, открывающая встроенный редактор напрямую.
- В fullscreen-плеере заголовок обновляется при переходе к следующему/предыдущему файлу, а дополнительные кнопки скорости, дорожек, субтитров и эквалайзера объединяются в одно меню на узких экранах.
- Под строкой текущего пути добавлена тонкая полоса памяти расположения с общим/занятым/свободным объемом для локальных дисков; виртуальные расположения показывают понятный статус недоступности такой информации.
- Сохранение позиции воспроизведения разделено настройками для видео и аудио; если до конца осталось меньше 10 секунд, файл считается досмотренным и позиция очищается.
- Добавлен предустановленный плагин `RAR Archive Support`: `.rar`, `.cbr`, `.rev` открываются как папки, файлы внутри доступны для просмотра, контекстное `Распаковать` работает для ZIP/RAR. На Android используется `rar`/libarchive FFI; на Windows/Linux используется CLI-компонент `7z/7za/unrar/rar` из папки RAR-плагина или PATH без Flutter native-assets hook.
- Добавлены предустановленные виртуальные плагины `RAID0 combined location` и `RAID1 mirror location`: через `Расположения +` можно собрать несколько уже настроенных расположений в одно, включая вложенные RAID0/RAID1; RAID0 распределяет файлы по свободному месту-подсказке или стабильному хэшу пути, RAID1 зеркалирует запись на все участники.
- Диалог создания профиля для RAID показывает чекбоксы существующих расположений-участников, а не требует ручного ввода технических id.
- Шаблоны Яндекс.Диск WebDAV и Nextcloud WebDAV сохранены как предустановленные реальные профили-исполнители с авторизацией через логин/app-password и настройкой через профиль подключения.
- Обновлены русские и английские встроенные строки и JSON-языки, исправлены новые строки в `ru.json`, которые могли отображаться с неправильной кодировкой.
- Выполнен `flutter analyze` после внесенных изменений.

## Реализовано в 0.12.3

- Версия повышена до `0.12.3` в `pubspec.yaml`, коде приложения и README.
- Для больших файлов добавлено блочное шифрование контейнеров `.crypt`: исходный файл больше не читается в память целиком, payload пишется чанками по 2 МБ, прогресс обновляется по байтам, операцию можно отменить через фоновую задачу. Это устраняет сбой шифрования крупных файлов на Android, который проявлялся на файлах порядка сотен мегабайт.
- В формат `.crypt` добавлен режим `payloadEncoding: chunked-v1`; старый single-box режим сохранен для малых файлов и обратной совместимости.
- Расшифрование блочных контейнеров поддержано в существующем просмотре/выгрузке, а имена и параметры зашифрованных файлов читаются только из заголовка контейнера без загрузки всего payload.
- Инспекция больших `.crypt` в проводнике переведена на чтение фиксированного заголовка, поэтому открытие папки с крупными зашифрованными файлами меньше нагружает память и UI.
- Создание файлов в remote/SMB расположениях переведено на `RemoteFileSystemClient.writeBytes`: обычные и сгенерированные зашифрованные файлы теперь создаются в SMB/FTP/SFTP/WebDAV профилях через удаленный executor, а не через локальный `File(remote://...)` путь.
- Текстовый редактор адаптирован под вертикальный телефон и узкие окна: основная область занимает ширину экрана, а панель конвертации/сохранения вызывается справа выдвижной панелью.
- Диалог добавления/редактирования профиля подключения адаптирован под узкие экраны: на телефоне открывается полноэкранная форма, endpoint-адреса `host/port` редактируются вертикальными карточками, кнопки сохранения вынесены в компактные иконки.
- В настройках поле масштаба интерфейса стало общим масштабом: допустимое уменьшение расширено до `0.55`, масштаб влияет на общий текст, размеры файлового списка и иконки проводника.
- Для элементов слева добавлены контекстные действия по долгому нажатию: недавние можно открыть или удалить из списка, избранное открыть или удалить, расположения открыть, а для профильных расположений также редактировать или удалить профиль.
- Панель фоновых задач стала плавающей: ее можно перемещать, сворачивать в круговой индикатор состояния и разворачивать обратно; в развернутом виде остались прогресс, статус, отмена и удаление завершенных задач.
- В окне `О программе` добавлена кнопка открытия файла логов программы; лог хранится в защищенной области данных приложения в папке `logs`.
- Перепроверены уже реализованные пункты профилей расположений: в `Все расположения` для профильных записей доступны `Редактировать профиль` и `Удалить профиль`, а состояние подключения отображается зеленым/красным статусом.
- Выполнены `flutter analyze`, `flutter test` и временный Flutter roundtrip-тест блочного `.crypt` контейнера размером больше порога блочного шифрования.

## Реализовано в 0.12.2

- Версия повышена до `0.12.2` в `pubspec.yaml`, коде приложения и README.
- В экране `Все расположения` для записей, созданных из профилей подключения, добавлены пункты контекстного меню `Редактировать профиль` и `Удалить профиль`.
- Редактирование профиля открывает тот же диалог профиля подключения с текущим названием, параметрами и списком endpoint-адресов `host/port`; изменения сохраняются в зашифрованной конфигурации и сразу обновляют список расположений.
- Удаление профиля запрашивает подтверждение, удаляет только профиль подключения, не удаляя плагин и не выполняя файловое удаление на удаленном ресурсе.
- Для записей-профилей в `Все расположения` отключены обычные потенциально опасные файловые действия вроде удаления/переименования корня ресурса; остаются безопасные действия открытия, избранного, свойств и управления профилем.
- `Все расположения` проверяет состояние подключения профилей через соответствующий remote executor с коротким timeout: доступные профили отмечаются зеленым, недоступные красным с текстом ошибки в подзаголовке.
- Добавлены метаданные `connectionProfileId` и `connectionStatus` в модель записи проводника, чтобы UI не угадывал профиль по строке пути.
- Обновлены русские и английские строки интерфейса для редактирования/удаления профиля и статуса подключения.
- `flutter analyze` и `flutter test` проходят без ошибок.

## Текущие технические границы

- Конфигурация сейчас шифруется встроенным device-secret ключом приложения. Полная схема `device-key first, app-password fallback` и перешифрование конфигурации паролем входа требует отдельной миграции формата настроек.
- Torrent streaming использует встроенный bencode/parser, локальный HTTP Range-сервер и управление torrent-разделом SecureVault. Piece-загрузку выполняет engine из папки torrent-плагина; Windows-компонент поставляется в `plugins/securevault_torrent/components/aria2/windows-x64/`, для других платформ используется соответствующий компонент плагина или development override `SECUREVAULT_ARIA2C`.
- Native Picture-in-Picture поверх других приложений, Android media notification service и глобальные headset-команды при полностью свернутом приложении требуют отдельного Android/Windows/Linux native service слоя.
- TrueCrypt/VeraCrypt-контейнеры распознаются как целевой формат, но полноценное монтирование требует системного драйвера или установленного VeraCrypt CLI; встроенный контейнер SecureVault сейчас реализован как encrypted ZIP.
- XLSX пока открывается как обычный документ/файл, без полноценного табличного редактора.
