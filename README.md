# Защищенное хранилище файлов

Версия приложения: `0.10.1`.

SecureVault - кроссплатформенный защищенный проводник для просмотра, шифрования, выгрузки, загрузки, редактирования и организации файлов. README ведется как карта проекта: сверху зафиксирован полный целевой функционал, ниже указано, что уже реализовано, какие ограничения остаются и что делать в следующих версиях.

## Полный целевой функционал

- Интерфейс как у проводника: расположения слева, файлы/папки по центру, область просмотра справа, изменяемая ширина областей, скрываемая область просмотра.
- Расположения: диски Windows, корень Linux/Android, стандартная область файлов телефона Android, скрытая папка программы, SMB/SSH/FTP/SFTP, облачные хранилища через JSON-плагины.
- Данные программы по умолчанию хранятся рядом с программой в `SecureVaultData`; в настройках можно переключить хранение в папку пользователя.
- Экспорт конфигурации, настроек, языков, плагинов и служебных файлов в ZIP-архив.
- Плагины: облачные хранилища, потоковые сайты, API/парсинг аудио и видео, поиск, плейлисты, подборки, обложки, версии, ссылки обновления, кроссплатформенные компоненты для разных ОС/архитектур, прокси на уровне программы, всех плагинов и отдельного плагина.
- Шаблоны плагинов: Яндекс.Диск WebDAV и Nextcloud WebDAV; целевой executor должен выполнять list/info/stream/upload/free-space и обновления.
- Объединение нескольких облачных хранилищ как одного расширяемого расположения с распределением файлов по свободному месту и оптимизацией размещения.
- Загрузка/выгрузка файлов как в проводнике: encrypted/plain, удаление источника после операции.
- Контекстные меню файлов, папок и пустой области: открыть, создать, создать папку, копировать, вырезать, вставить, удалить, переименовать, свойства, отправить, ZIP-операции, шифровать/расшифровать, использовать папку как галерею/видео/аудио/мультимедиа.
- Создание обычных и зашифрованных файлов, CSV-таблиц, изображений; для CSV выбирается разделитель, для изображения формат, размер, фон и прозрачность.
- Шифрование файлов общим или уникальным ключом, выбор алгоритма, соли/параметры, ключ строкой или файлом; общий ключ настраивается в настройках.
- Пароль входа и отдельный пароль зашифрованных файлов, сохранение паролей в зашифрованном виде, очистка сохраненных паролей при ошибках входа.
- Зашифрованный конфигурационный файл; целевой режим - при пароле входа шифровать конфигурацию этим паролем, иначе встроенным ключом.
- Автоматическое отображение расшифрованных имен файлов и папок при доступном ключе; настройка отключения.
- Недавние и избранные файлы/папки в левой панели с настройкой количества, проверкой доступности и удалением отсутствующих записей.
- Поиск по имени, содержимому или имени+содержимому, регулярные выражения, поиск в подпапках, фильтры по размеру/датам/длительности.
- Просмотр документов, текста, HTML, изображений, аудио и видео обычных и зашифрованных файлов; зашифрованное содержимое должно обрабатываться в памяти и не раскрываться в постоянное хранилище.
- Аудио/видео плейлист внизу области просмотра, повтор, случайное воспроизведение, переход к следующему файлу, запоминание позиции остановки.
- Фоновое воспроизведение: мини-видео и мини-аудиоплеер, перетаскивание по окну, сворачивание/разворачивание/закрытие, настройки отключения фонового поведения.
- Галерея как на телефоне: изображения и видео по годам/месяцам, миниатюры, масштабирование Ctrl+колесико до календарного вида.
- Музыка и Видео как медиатеки: вкладки Все, Плейлисты, Альбомы, Исполнители, Папки; обложки аудио из метаданных.
- Торрент-раздел: добавление `.torrent`, хранение в папке программы, просмотр как папки, потоковое воспроизведение и выборочная загрузка.
- Редакторы отдельными окнами: изображение со слоями, crop/rotate/draw/erase/import layer; аудио с waveform/trim/glue/mix; видео с timeline/trim/concat/resize/rotate/quality/audio track/frame layers.
- Многоязычность: русский и английский по умолчанию, импорт языковых JSON-файлов, экспорт английского образца.
- Защита от скриншотов/записи экрана там, где это позволяет платформа; предупреждение при открытии через внешние приложения.
- Android: запрос прав на файлы/медиа, запоминание отказа, повторный запрос из настроек.
- Сборки Windows, Android и Linux; Linux release собирается на Linux-хосте.

## Реализовано к версии 0.10.0

- Версия повышена до `0.10.0`.
- Подключен реальный слой сетевых и облачных исполнителей `remote://`: WebDAV/JSON HTTP для Яндекс.Диск и Nextcloud, нативный FTP-клиент на Dart с пассивным режимом, SFTP через системный `ssh`, SMB через `smbclient`.
- Плагины WebDAV дополнены переменными окружения для логина, пароля и базового URL; добавлены шаблоны `ftp-resource`, `sftp-resource`, `smb-resource` с capability `listFiles`, `fileInfo`, `fileStream`, `upload`, `mkdir`, `delete`.
- Облачные и сетевые плагины, для которых есть поддерживаемый executor, теперь появляются в расположениях как активные виртуальные папки и открываются прямо в проводнике.
- `.torrent` теперь разбирается встроенным bencode-парсером, открывается как виртуальная папка `torrent://...`, показывает вложенную структуру файлов и запускает выборочную загрузку через `aria2c` для выбранного файла.
- Медиа-плеер вынесен в общий долгоживущий service: один `media_kit` player переживает переходы между разделами, сохраняет общий playlist/session и показывает плавающее мини-видео или мини-аудиоплеер с перетаскиванием, сворачиванием, разворачиванием и закрытием.
- In-memory loopback streaming adapter перенесен в общую media-сессию: расшифрованные и ZIP-медиа по-прежнему отдаются плееру из оперативной памяти с поддержкой `Range`.
- В аудио/видео-редактор добавлен графический waveform/timeline-блок с `RangeSlider`; `ffprobe` используется для определения длительности, а выбранный диапазон автоматически заполняет поля trim start/end для `ffmpeg`.
- `flutter analyze` проходит без ошибок после добавления remote/torrent/media/editor слоев.

## Реализовано к версии 0.9.0

- Версия повышена до `0.9.0`.

- Для навигации в недоступные расположения реализовано поведение `ask`, `deny`, `allow`, `fallbackToLocations`, `requestRoot` с пользовательским предупреждением там, где это применимо.

## Ограничения текущей версии

- Полное шифрование конфигурации паролем входа еще не завершено: сейчас настройки шифруются встроенным ключом приложения. Переключение папки хранения уже работает, но режим `device-key first, app-password fallback` остается следующим этапом.
- Executor сетевых/облачных плагинов подключен для WebDAV/FTP/SFTP/SMB. Стриминговые site/API-плагины, OAuth HTML-return и автоматическое обновление плагинов остаются следующим этапом.
- Фоновый мини-плеер вынесен в общий player-сервис внутри окна приложения. Системный Android/Windows Picture-in-Picture поверх других приложений пока не подключен native-слоем.
- Для расшифрованного preview аудио/видео используется in-memory loopback streaming adapter без временных файлов SecureVault и теперь он работает через общий player-сервис. Следующий этап - потоковое чтение больших файлов чанками без полной загрузки в память.
- Создание зашифрованного контейнера из папки реализовано как encrypted ZIP-контейнер SecureVault. TrueCrypt/VeraCrypt требуют отдельного mount-adapter через системный драйвер или установленный VeraCrypt CLI и пока не монтируются внутри приложения.
- Фильтры поиска по размеру/датам подключены к текущему списку проводника; фильтры длительности сохранены в интерфейсе для следующего этапа с медиаиндексом.
- Галерея уже группируется по годам/месяцам и имеет календарный zoom-режим; следующие улучшения - быстрые переходы по датам, сортировка по EXIF/медиаметаданным и кэш миниатюр.
- Музыка/Видео пока не имеют вкладок Альбомы/Исполнители/Папки на основе медиаметаданных.
- Редакторы аудио/видео получили графический waveform/timeline trim-слой поверх `ffmpeg`. Покадровые видеослои и полноценная нелинейная монтажная модель остаются следующим этапом.
- Прогресс рекурсивного шифрования/расшифрования в отдельном потоке с отменой еще не подключен.
- Torrent adapter умеет читать `.torrent`, показывать файлы как папку и запускать выборочную загрузку через `aria2c`. Полный встроенный BitTorrent-движок без внешнего `aria2c` и настоящий piece-streaming до завершения загрузки остаются следующим этапом.
- Linux release не собирается в этой Windows-среде; его нужно собирать на Linux-хосте с Linux desktop toolchain.
