# Защищенное хранилище файлов

Версия приложения: `0.6.0`.

SecureVault - кроссплатформенный проводник для защищенного хранения, просмотра, шифрования и редактирования файлов. README ведется как карта проекта: сверху зафиксирован полный целевой функционал, ниже - что уже реализовано, текущие ограничения и план следующей версии.

## Полный целевой функционал

- Интерфейс как у проводника: переход по папкам, область расположений, область файлов и область просмотра с изменяемой шириной.
- Расположения: диски Windows, корень Linux/Android, стандартная область файлов телефона на Android, скрытая папка приложения, сетевые ресурсы SMB/SSH/FTP/SFTP, облачные хранилища через подключаемые JSON-плагины.
- Избранное и недавние файлы над расположениями, отдельный режим просмотра последних файлов с проверкой доступности.
- Загрузка и выгрузка как в проводнике: encrypted/plain, удаление исходника после операции.
- Контекстные меню для файлов и папок: открыть, копировать, вырезать, вставить, удалить, переименовать, свойства, отправить, архивировать, шифровать/расшифровать, операции над папками.
- Создание обычных и зашифрованных файлов, CSV-таблиц и изображений.
- Шифрование файлов общим ключом или уникальным ключом с выбором алгоритма, соли и параметров; общий ключ может задаваться текстом или файлом-ключом.
- Пароль входа в приложение и отдельный пароль для зашифрованных файлов; смена пароля требует текущий пароль, ввод текущего без нового снимает пароль.
- Настройки и сохраненные секреты должны храниться в зашифрованном виде; целевой режим - при наличии пароля входа шифровать конфигурацию этим паролем.
- Автоматическое отображение расшифрованных имен файлов и папок, если доступный пароль/общий ключ позволяет это сделать.
- Просмотр документов, изображений, аудио, видео, HTML и архивов в области просмотра; для зашифрованных файлов просмотр должен по возможности не писать расшифрованное содержимое в постоянное хранилище.
- Аудио/видео плейлист по файлам текущей папки, случайное воспроизведение, повтор одного файла, цикл списка и запоминание позиции остановки.
- Открытие просмотра во все окно и открытие другой программой только после предупреждения о раскрытии данных.
- Детектирование или блокировка скриншотов/записи экрана там, где это позволяет платформа.
- Многоязычность: русский и английский по умолчанию, импорт пользовательских JSON-языков, экспорт образца для перевода.
- Зашифрованные контейнеры собственного формата, понимание/монтирование контейнеров TrueCrypt/VeraCrypt.
- Встроенные редакторы изображений, аудио и видео, включая покадровые/слойные сценарии для видео.
- Подключаемые облачные плагины, Яндекс.Диск/Nextcloud, объединенный облачный диск с распределением файлов и оптимизацией свободного места.
- Torrent-раздел с потоковым воспроизведением выбранных файлов.
- Сборки Windows, Android и Linux, ведение git-версий.

## Реализовано к версии 0.6.0

- Flutter-интерфейс проводника с левой панелью расположений, списком файлов и областью просмотра.
- Изменяемая ширина левой панели и области просмотра; область просмотра можно скрыть и раскрыть шире прежнего ограничения.
- Вторичные действия области просмотра вынесены в меню с тремя точками; кнопка просмотра называется `Открыть во все окно`.
- Кнопки загрузки и выгрузки находятся сверху проводника и отображаются как иконки.
- Поддержаны локальные расположения Windows, корневые расположения Linux/Android, файловая область Android и скрытая папка приложения.
- Зарезервированы провайдеры SMB, SSH/SCP, FTP, SFTP и JSON cloud-plugin; JSON-манифесты облачных плагинов обнаруживаются в папке приложения.
- Встроены шаблоны JSON-плагинов для Яндекс.Диск WebDAV и Nextcloud WebDAV; ZIP-плагины можно устанавливать через настройки.
- Загрузка/выгрузка умеет копировать plain-файлы, создавать `.crypt`, экспортировать encrypted-файл или расшифрованную копию по выбору пользователя.
- Для app-created `.crypt` реализованы XChaCha20-Poly1305 и AES-256-GCM с Argon2id-derived ключом, солью и метаданными режима ключа.
- Общий ключ шифрования хранится в зашифрованной оболочке; при отсутствии отдельного пароля зашифрованных файлов используется автоматически без запроса.
- Текущий общий ключ показывается без запроса пароля, если не настроены ни пароль входа, ни отдельный пароль зашифрованных файлов.
- Пароль входа и пароль зашифрованных файлов меняются через поля текущий/новый/повтор; ввод текущего без нового снимает пароль.
- Добавлена поддержка файла-ключа для общего шифрования.
- Настройки сохраняются не открытым JSON, а зашифрованной оболочкой на встроенном ключе приложения; старый открытый JSON читается для совместимости.
- Сохраненный пароль зашифрованных файлов хранится в зашифрованной оболочке и может очищаться при неправильном входе.
- Настраивается таймаут, в течение которого пароль зашифрованных файлов не требуется повторно в активной сессии.
- Добавлены избранные файлы и папки, недавние файлы, настройки количества элементов и удаление недоступных записей.
- Контекстное меню расширено: открыть, создать, копировать, вырезать, вставить, удалить, переименовать, свойства, отправить, добавить/удалить избранное, удалить из недавних, ZIP-распаковка, шифровать/расшифровать файл.
- Реализованы локальные операции копирования/вырезания/вставки, удаления, переименования и просмотра свойств.
- В меню папки добавлены целевые кнопки `Создать зашифрованный контейнер из папки`, `Зашифровать файлы`, `Расшифровать файлы`; рекурсивная крипто-логика пока запланирована отдельно.
- Создание файлов из контекстного меню папки: обычный файл, зашифрованный файл, CSV, зашифрованный CSV, изображение, зашифрованное изображение.
- Для CSV задается разделитель; для изображений задаются PNG/JPG/BMP, размер, цвет фона и прозрачность.
- Просмотр расшифрованных app-created `.crypt` выполняется в оперативной памяти; постоянная запись выполняется только при явной выгрузке/расшифровке или внешнем открытии после предупреждения.
- Реализован просмотр текста, логов, исходников, изображений, HTML text preview, ZIP-содержимого и извлечение текста из PDF/DOCX/ODT/XLSX/PPTX/RTF где возможно.
- Аудио и видео открываются во встроенном просмотрщике на `media_kit`: есть плейлист по файлам текущей папки, переключение на следующий файл, цикл по списку, случайное воспроизведение и повтор текущего файла.
- Зашифрованные аудио/видео, созданные программой, проигрываются после расшифровки активным паролем/общим ключом через защищенный сеанс просмотра.
- Двойное нажатие по файлу открывает просмотр во все окно; если область просмотра скрыта, одиночное нажатие может открывать файл во все окно, опция включена по умолчанию.
- Для изображений добавлено перелистывание соседних картинок в папке стрелками влево/вправо и горизонтальным свайпом; работает и в области просмотра, и при открытии во все окно.
- Раздел `Галерея` показывает найденные изображения и видео плитками с миниатюрами; для Android видео-миниатюра берется из первого кадра, на остальных платформах используется безопасный fallback.
- Разделы `Музыка` и `Видео` работают как медиатеки без области проводника: сразу открывается проигрыватель с плейлистом всех найденных аудио/видео из настроенных или доступных путей.
- При проигрывании аудио показывается встроенная обложка: на Android через `MediaMetadataRetriever`, для MP3 дополнительно через разбор ID3/APIC в Dart.
- На Android добавлена проверка доступа к медиа/файлам, запрос runtime-разрешений и переход в системную настройку доступа ко всем файлам; если пользователь отказался в диалоге приложения, повторно автоматически не спрашиваем, но в настройках есть кнопка повторного запроса.
- Добавлен редактор изображений: сохранение в PNG/JPG/BMP, поворот, центральная обрезка, рисование поверх изображения, простой ластик, отмена и очистка нарисованных слоев.
- Добавлен базовый аудио/видео-редактор через `ffmpeg`-адаптер: обрезка по времени, склейка с другим файлом, добавление/замена звука, изменение размера/поворот видео, настройка CRF видео и битрейта аудио, сохранение в другой формат по расширению выходного файла.
- В левой панели добавлены разделы `Галерея`, `Музыка`, `Видео`, `Документы`, `Торрент`; в настройках добавлены папки и исключения для медиа-разделов.
- Нажатие `Проводник` показывает существующие расположения и настроенные пути как стартовый список.
- Строка пути редактируется двойным нажатием; перед переходом программа проверяет доступность пути.
- Добавлен поиск по текущему списку файлов и зарезервирован диалог фильтров по длительности/датам/размеру.
- В настройках добавлены размеры текста/иконок файлов и автоматическая подстройка по DPI устройства.
- Для app-created `.crypt`, доступных по общему ключу или активному файловому паролю, проводник может показывать расшифрованное имя; опцию можно отключить.
- Языковые JSON-файлы можно устанавливать в папку программы; русский и английский файлы обновлены.
- Android использует `FLAG_SECURE` для блокировки скриншотов/записи экрана там, где это поддерживается ОС.
- Desktop/Android open-with входные пути маршрутизируются в просмотр, если платформа предоставила читаемый файл.
- Нативное C17-ядро `crypt_core` собирается и проверяет формат контейнеров/метаданных через FFI.
- Логотип Windows/Android заменен на файлы из папки `icons`; заголовок окна Windows изменен с `secure_vault_shell` на название программы.
- Исправлена кодировка Windows-заголовка: нативный runner использует Unicode-строку, а Flutter после загрузки языка передает заголовок окна через Windows method channel.
- Flutter `analyze`, smoke-тест и тест round-trip шифрования app-created `.crypt` проходят.
- Windows и Android release 0.6.0 собраны.

## Ограничения текущей версии

- Полное шифрование конфигурации паролем входа еще не завершено: сейчас файл настроек шифруется встроенным ключом приложения, а целевой режим с запросом пароля входа до чтения конфигурации запланирован на следующую версию.
- Операции папки `создать зашифрованный контейнер`, `зашифровать файлы`, `расшифровать файлы` добавлены в меню, но рекурсивная крипто-логика еще не подключена.
- Собственные монтируемые контейнеры и совместимость с TrueCrypt/VeraCrypt пока находятся в проектном списке; для реального монтирования потребуются платформенные драйверы/адаптеры.
- Запоминание позиции остановки аудио/видео еще не подключено к настройкам/истории.
- Видеоредактор пока не содержит покадровый редактор слоев внутри таймлайна; базовые операции выполняются через `ffmpeg`, а покадровая работа запланирована отдельным модулем.
- Аудио/видео-редактор требует установленный `ffmpeg` в `PATH`; если `ffmpeg` отсутствует, диалог сообщает об этом.
- Linux release не собран в этой Windows-среде: Flutter desktop Linux требует Linux toolchain и сборку на Linux-хосте.
- Сетевые протоколы и облачные плагины пока являются подготовленными точками расширения без транспортного executor.
- Torrent-раздел подготовлен, но потоковое воспроизведение torrent-файлов требует отдельный torrent adapter.
- На Windows/Linux миниатюры видео пока показываются fallback-иконкой, если нет платформенного thumbnail adapter; Android уже извлекает первый кадр.
- Android-доступ ко всем файлам зависит от решения пользователя и системной политики ОС; если право не выдано, приложение показывает только доступные области.

- Подключить рекурсивное шифрование/расшифрование папок и создание собственного зашифрованного контейнера из папки.
- Добавить сохранение позиции остановки аудио/видео и восстановление позиции при следующем открытии.
- Расширить редакторы: недеструктивные слои изображений, импорт изображения как редактируемого слоя, waveform/timeline для аудио и покадровые слои для видео.
- Спроектировать собственный контейнер и платформенный mount-adapter; отдельно исследовать совместимость TrueCrypt/VeraCrypt.
- Реализовать SMB/SSH/FTP/SFTP, WebDAV/cloud executor и объединенный облачный диск с распределением файлов.
- Реализовать torrent adapter с потоковым воспроизведением выбранных файлов.
- Собрать Linux release на Linux-хосте.

## Проверка и сборка

Нативное ядро:

```powershell
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --config Release
ctest --test-dir build -C Release --output-on-failure
```

Flutter:

```powershell
cd app/flutter_shell
..\..\tooling\flutter\flutter\bin\flutter.bat analyze
..\..\tooling\flutter\flutter\bin\flutter.bat test
..\..\tooling\flutter\flutter\bin\flutter.bat build windows --release
..\..\tooling\flutter\flutter\bin\flutter.bat build apk --release
```

## Артефакты релиза

После сборки подготовленные файлы находятся в `dist/`:

- `SecureVault-android-0.6.0-release.apk`
- `SecureVault-windows-x64-0.6.0.zip`
- `SHA256SUMS.txt`

Android APK подписан debug-конфигурацией Flutter и подходит для тестовой установки, но не для публикации в Google Play.
