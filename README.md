# Защищенное хранилище файов

Версия приложения: `0.3.0`.

SecureVault - кроссплатформенный проводник для защищенного хранения, просмотра и передачи файлов. README ведется как карта проекта: здесь зафиксирован полный целевой функционал, реализованное состояние, ограничения текущей версии и задачи следующей итерации.

## Полный целевой функционал

- Интерфейс как у проводника: переход по папкам, область расположений, область файлов и область просмотра с изменяемой шириной.
- Расположения: диски Windows, корень Linux/Android, стандартная область файлов телефона на Android, скрытая папка приложения, сетевые ресурсы SMB/SSH/FTP/SFTP, облачные хранилища через подключаемые JSON-плагины.
- Избранное: добавление файлов и папок в левую панель над расположениями.
- Недавние: список последних открытых файлов в левой панели и отдельный режим просмотра последних файлов с проверкой доступности.
- Загрузка и выгрузка как в проводнике: encrypted/plain, удаление исходника после операции.
- Контекстные меню для файлов и папок: открыть, копировать, вырезать, вставить, удалить, переименовать, свойства, шифрование/расшифрование, операции над папками.
- Шифрование файлов общим ключом или уникальным ключом с выбором алгоритма, соли и параметров.
- Общий ключ может задаваться текстом или файлом-ключом.
- Автоматическое использование общего ключа без повторного вопроса, если отдельный пароль зашифрованных файлов не включен.
- Пароль входа в приложение и отдельный пароль для зашифрованных файлов: можно использовать один пароль или разные пароли.
- Смена пароля требует текущий пароль; ввод текущего пароля без нового снимает соответствующий пароль.
- Настройки и сохраненные секреты должны храниться в зашифрованном виде; при наличии пароля входа целевое поведение - шифровать конфигурацию этим паролем.
- Автоматическое отображение расшифрованных имен файлов в проводнике, если политика доступа разрешает расшифровку.
- Просмотр документов, изображений, аудио и видео в области просмотра без записи расшифрованных данных в постоянную память.
- Аудио/видео плейлист по файлам текущей папки и запоминание позиции остановки просмотра.
- Открытие просмотра во все окно.
- Открытие другой программой только после предупреждения о возможном раскрытии данных.
- Детектирование или блокировка скриншотов/записи экрана там, где это позволяет платформа.
- Многоязычность: русский и английский по умолчанию, импорт пользовательских JSON-языков, экспорт образца для перевода.
- Зашифрованные контейнеры собственного формата, а также понимание/монтирование контейнеров TrueCrypt/VeraCrypt.
- Встроенные редакторы изображений, аудио и видео.
- Сборки Windows, Android и Linux, ведение git-версий.

## Реализовано в версии 0.3.0

- Flutter-интерфейс проводника с левой панелью расположений, списком файлов и областью просмотра.
- Изменяемая ширина левой панели и области просмотра; область просмотра можно скрыть.
- Вторичные действия области просмотра вынесены в меню с тремя точками.
- Кнопки загрузки и выгрузки находятся сверху проводника.
- Шифрование и расшифрование файлов доступны из контекстного меню файла.
- Поддержаны локальные расположения Windows, корневые расположения Linux/Android, файловая область Android и скрытая папка приложения.
- Зарезервированы провайдеры SMB, SSH/SCP, FTP, SFTP и JSON cloud-plugin; JSON-манифесты облачных плагинов обнаруживаются в папке приложения.
- Загрузка/выгрузка умеет копировать plain-файлы, создавать `.crypt`, экспортировать encrypted-файл или расшифрованную копию по явному выбору пользователя.
- Для app-created `.crypt` реализованы XChaCha20-Poly1305 и AES-256-GCM с Argon2id-derived ключом, солью и метаданными режима ключа.
- Общий ключ шифрования хранится в зашифрованной оболочке; при отсутствии отдельного пароля зашифрованных файлов используется автоматически без запроса у пользователя.
- Текущий общий ключ показывается без запроса пароля, если не настроены ни пароль входа, ни отдельный пароль зашифрованных файлов.
- Если пароль входа или отдельный пароль файлов настроен, показ общего ключа требует соответствующий пароль.
- Пароль входа и пароль зашифрованных файлов меняются через поля текущий/новый/повтор; ввод текущего без нового снимает пароль.
- Добавлена поддержка файла-ключа для общего шифрования: если поле общего пароля пустое, ключ может быть получен из содержимого указанного файла.
- Настройки сохраняются не открытым JSON, а зашифрованной оболочкой на встроенном ключе приложения; старый открытый JSON продолжает читаться для совместимости.
- Сохраненный пароль зашифрованных файлов хранится в зашифрованной оболочке и может очищаться при неправильном входе.
- Настраивается таймаут, в течение которого пароль зашифрованных файлов не требуется повторно в активной сессии.
- Добавлены избранные файлы и папки: пункт контекстного меню добавляет/удаляет путь, левая панель показывает избранное над расположениями.
- Добавлены недавние файлы: приложение запоминает открытые файлы, показывает последние элементы слева и открывает виртуальную папку `Недавние`.
- В настройках можно отключить запоминание недавних файлов, задать количество недавних слева и количество элементов истории.
- При попытке открыть недавний файл, которого больше нет, программа сообщает об этом и предлагает удалить запись из списка.
- Контекстное меню расширено: открыть, копировать, вырезать, вставить, удалить, переименовать, свойства, добавить/удалить избранное, удалить из недавних, шифровать/расшифровать файл.
- Реализованы локальные операции копирования/вырезания/вставки, удаления, переименования и просмотра свойств для файлов и папок.
- В меню папки добавлены целевые кнопки `Создать зашифрованный контейнер из папки`, `Зашифровать файлы`, `Расшифровать файлы`; рекурсивная крипто-логика пока запланирована отдельно.
- Просмотр расшифрованных app-created `.crypt` выполняется в оперативной памяти; постоянная запись выполняется только при явной выгрузке/расшифровке или внешнем открытии после предупреждения.
- Реализован просмотр текста, логов, исходников, изображений и извлечение текста из PDF/DOCX/ODT/XLSX/PPTX/RTF где возможно.
- Аудио и видео распознаются, показывается защищенная мета-информация; нативное проигрывание пока не подключено из-за ограничения Flutter-плагинов в текущей Windows-среде без symlink support.
- Кнопка просмотра называется `Открыть во все окно`.
- В левой верхней части показано только название программы; сведения о версии и нативном ядре перенесены в круглую кнопку `i` (`О программе`).
- Русский и английский интерфейсы синхронизированы; можно валидировать пользовательский JSON-язык и выгрузить английский образец.
- Android использует `FLAG_SECURE` для блокировки скриншотов/записи экрана там, где это поддерживается ОС.
- Desktop/Android open-with входные пути маршрутизируются в просмотр, если платформа предоставила читаемый файл.
- Нативное C17-ядро `crypt_core` собирается и проверяет формат контейнеров/метаданных через FFI.
- Flutter smoke-тест и тест round-trip шифрования app-created `.crypt` проходят.

## Ограничения текущей версии

- Полное шифрование конфигурации паролем входа еще не завершено: сейчас файл настроек шифруется встроенным ключом приложения, а целевой режим с запросом пароля входа до чтения конфигурации запланирован на следующую версию.
- Автоматическая расшифровка имен файлов в списке проводника пока не реализована полностью; контейнерные параметры читаются для просмотра и расшифровки.
- Операции папки `создать зашифрованный контейнер`, `зашифровать файлы`, `расшифровать файлы` добавлены в меню, но рекурсивная крипто-логика еще не подключена.
- Собственные монтируемые контейнеры и совместимость с TrueCrypt/VeraCrypt пока находятся в проектном списке; для реального монтирования потребуются платформенные драйверы/адаптеры.
- Нативный аудио/видео просмотр, плейлисты и запоминание позиции остановки еще не реализованы.
- Редакторы изображений, аудио и видео еще не реализованы.
- Linux release не собран в этой Windows-среде: Flutter desktop Linux требует Linux toolchain и сборку на Linux-хосте.
- Сетевые протоколы и облачные плагины пока являются подготовленными точками расширения без транспортного executor.

## Следующая версия

- Переработать загрузку настроек на `device-key first, app-password fallback`: если встроенный ключ не расшифровывает конфигурацию, показать экран ввода пароля входа и расшифровать настройки им.
- При сохранении настроек с включенным паролем входа запрашивать текущий пароль и перешифровывать конфигурацию паролем входа.
- Добавить расшифровку отображаемых имен `.crypt` в списке файлов при доступном общем ключе/пароле.
- Подключить рекурсивное шифрование/расшифрование папок и создание собственного зашифрованного контейнера из папки.
- Подключить безопасный нативный медиаплеер без записи расшифрованных файлов в постоянное хранилище; добавить плейлист и запоминание позиции.
- Спроектировать собственный контейнер и платформенный mount-adapter; отдельно исследовать совместимость TrueCrypt/VeraCrypt.
- Добавить редакторы изображений, аудио и видео.
- Реализовать SMB/SSH/FTP/SFTP и executor для JSON cloud-plugin.
- Собрать Linux release на Linux-хосте.

## Проверка и сборка

Нативное ядро:

```powershell
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --config Release
ctest --test-dir build -C Release --output-on-failure
```

Flutter:

```powershell
cd app/flutter_shell
..\..\tooling\flutter\flutter\bin\flutter.bat analyze
..\..\tooling\flutter\flutter\bin\flutter.bat test
..\..\tooling\flutter\flutter\bin\flutter.bat build windows --release
..\..\tooling\flutter\flutter\bin\flutter.bat build apk --release
```

## Артефакты релиза

После сборки подготовленные файлы находятся в `dist/`:

- `SecureVault-windows-x64-0.3.0.zip`
- `SecureVault-android-0.3.0-release.apk`
- `SHA256SUMS.txt`

Android APK подписан debug-конфигурацией Flutter и подходит для тестовой установки, но не для публикации в Google Play.
